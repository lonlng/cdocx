## 1. 执行摘要

CDocx 是一个基于 C++17 的 DOCX 操作库，当前版本 **v0.8.0**。DOM 架构的核心基础设施已全部闭环，Legacy Iterator API 稳定可用，`src/` 目录内所有 TODO 占位均已清零。**19 组测试用例全部通过**，代码格式化与静态分析均通过 CI 检查。

**当前状态：**

2. **DOM 同步机制完整闭环**：`sync_sections_to_physical()` 和 `sync_sections_from_physical()` 支持 Run、Paragraph、Table、Section、Header/Footer、Bookmark、Field、Hyperlink、Shading、DropCap 的序列化与反序列化。
3. **L2 中间功能全部落地**：DocumentSearch、TableOperations（merge/split、列插入删除、auto_fit）、DocumentBuilder（hyperlink、image、bookmark、footnote）、StyleCollection、MailMerge、Watermark、CommentCollection 均已实现并通过测试。

## 3. 功能需求分析

### 3.1 API 路径说明

| 路径 | 抽象层级 | 适用场景 | 当前状态 |
|------|---------|---------|---------|
| **DOM API** | 高 | 创建/编辑文档内容，无需了解 Open XML | **完整可用**，所有核心节点均已实现序列化/反序列化 |
| **XML Parts API** | 低 | 直接修改 styles、settings、relationships 等底层 XML | **完整可用**，修改后自动写回 ZIP |
| **Legacy Iterator API** | 低（兼容层） | 旧代码维护，基于 XML 节点的游标式遍历 | **稳定可用**，但不与 DOM 树同步 |

### 3.2 功能需求三层模型

| 层级 | 需求描述 | 关键实现 | 完成度 |
|------|---------|---------|--------|
| **L1 基础功能** | 文档生命周期、DOM 树增删改查、基础格式、页面结构、列表 | `Document::open/save/create_empty`、DOM 节点体系、`Paragraph`/`Run`/`Table` CRUD、`Section`/`HeaderFooter`、列表系统 | **~100%** |
| **L2 中间功能** | 模板替换、书签替换、文档合并、媒体管理、表格构建、表格合并拆分、列插入删除、auto_fit、全文搜索替换、DOM↔XML 同步、节点克隆与遍历、DocumentBuilder、样式集合、邮件合并、水印、批注 | `Template`、`BookmarkReplacer`、`DocumentInserter`、媒体 API、`TableBuilder`、`Table::merge_cells` / `split_cell` / `insert_column` / `delete_column` / `auto_fit`、`DocumentSearch`、`document_sync.cpp`、克隆/遍历、`DocumentBuilder`、`StyleCollection`、`MailMerge`、`Watermark`、`CommentCollection` | **~100%** |
| **L3 高级功能** | 复杂字段定制、脚注尾注集合 API、PDF/HTML 导出 | 基础 `Field` DOM 类及序列化已实现；**`Field` 字段开关（switches）定制及 DOM↔XML 双向同步已实现**；**`FootnoteCollection` / `EndnoteCollection` 及 DOM↔XML 双向同步已实现**；PDF/HTML 导出在 Roadmap 中 | **~80%** |
